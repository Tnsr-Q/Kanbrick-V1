# Probe — Upstream WASM / toolchain compatibility matrix (P8.1)

> Probe note — P8.1 (#93) · part of P8 — Upstream De-Risk (#79). Backs ADR-0014.

## What we set out to do

Initialize `crates/ironclaw` + `crates/tachyon-mesh`, build each in isolation, and
record their wasmtime / WASI / toolchain versions against our pinned **rustc
1.94.1 / wasmtime 45 / WASIp1**, to confirm the single-runtime decision.

## What happened in this environment

`git submodule update --init --depth 1 crates/ironclaw crates/tachyon-mesh` returns
**HTTP 403** from the agent proxy: only `tnsr-q/kanbrick-v1` is in network scope
for this de-risk session, so `JoasASantos/ironclaw` and `astorise/Tachyon-Mesh`
cannot be cloned here. The matrix below is reconstructed from **ADR-0002's recorded
probe** (run 2026-06-15, `core-host` @ `31d63b9`) and the **Phase-2 Ironclaw
finding**, and is reproduced on a submodule-capable machine / CI.

## Compatibility matrix

| Component | Role | Runtime / ABI | wasmtime | WASI / target | toolchain | In our build graph? |
| --- | --- | --- | --- | --- | --- | --- |
| `kanbrick-mesh` | **the** guest runtime | own minimal host↔guest ABI (JSON boundary) | 45 | WASIp1 / `wasm32-wasip1` | 1.94.1 | yes |
| tachyon-mesh `core-host` | reference only (~50K LOC / 76 files; eBPF `aya`, `candle`/ONNX) | Component Model + WIT | 45.0.0 | WASIp2 / `wasm32-wasip2` | 1.96.0 | **no** |
| ironclaw | primitives only (`jsonwebtoken`, `argon2`) | n/a — binary, no lib target | — | n/a | ≥ 1.75 | **no** |

## Conclusion

The single-runtime decision is confirmed (→ **ADR-0014**). Both submodules stay
**excluded** from the root workspace (`crates/ironclaw`, `crates/tachyon-mesh` in
`Cargo.toml` `exclude`, alongside `crates/sparrowdb`, `crates/sparrowdb-ontology`,
the `guests/*` wasm fixtures, `cockpit/src-tauri`, and `probes/*`), so `cargo build
--workspace` is unaffected. Re-running the init + isolated builds on a
network-capable machine reproduces the matrix; nothing about the decision depends
on values not already recorded in ADR-0002.
